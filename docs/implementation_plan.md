# PokeTutorial - オンボーディング実装計画 (5 Days)

この計画は、SpringBootとVue.jsの基礎を学びながら、5日間で「PokeCa Pocket Lite」を完成させるためのロードマップです。
各日は「学習目標」と「実装タスク」で構成されています。

## Day 1: 環境構築とHello World
**目標**: 開発環境を整え、バックエンドとフロントエンドが通信できる状態にする。

### Tasks
1. **プロジェクト作成**
   - [Backend] Spring InitializrでSpring Bootプロジェクトを作成する。
   - [Frontend] `npm create vue@latest` でVue.jsプロジェクトを作成する。
2. **Hello World APIの実装**
   - Spring Bootで `/api/hello` エンドポイントを作成し、文字列を返す。
3. **API呼び出し**
   - Vue.jsから `/api/hello` を呼び出し、画面に表示する。
   - CORS (Cross-Origin Resource Sharing) の設定を行う。

## Day 2: バックエンドロジック - PokeAPI連携
**目標**: 外部APIを利用してデータを取得し、加工して返すロジックを実装する。

### Tasks
1. **PokeAPIクライアントの実装**
   - `RestClient` または `RestTemplate` を使い、PokeAPIからポケモン情報を取得するServiceを作成する。
2. **データモデルの定義**
   - アプリ内で扱う `Card` クラス (DTO) を定義する (名前、画像URL、タイプなど)。
3. **パック開封ロジック**
   - ランダムなポケモンIDを5つ生成し、それぞれの情報を取得してリストで返す `/api/pack/open` エンドポイントを作成する。

## Day 3: フロントエンド - パック開封UI
**目標**: ユーザーが操作できる画面を作り、バックエンドのデータをリッチに表示する。

### Tasks
1. **開封ボタンの作成**
   - 「パックを開ける」ボタンを配置する。
2. **カードコンポーネントの作成**
   - `Card.vue` コンポーネントを作成し、ポケモンの画像と名前を表示できるようにする。
3. **開封結果の表示**
   - ボタンクリックで `/api/pack/open` を叩き、返ってきた5枚のカードを並べて表示する。

## Day 4: データベース連携 - コレクション機能
**目標**: データの永続化 (保存) を学び、アプリに「集める」要素を追加する。

### Tasks
1. **データベース設定**
   - H2 Databaseを有効化する。
   - `Card` エンティティを作成し、DBテーブルとマッピングする。
2. **保存ロジックの追加**
   - パック開封時に、取得したカード情報をDBに保存するようにバックエンドを修正する。
3. **コレクションAPIの実装**
   - 保存された全カードを取得する `/api/cards` エンドポイントを作成する。
4. **コレクション画面の作成**
   - Vue.jsで「マイコレクション」ページを作成し、所持カード一覧を表示する。

## Day 5: UI/UX改善と仕上げ
**目標**: アプリの見た目を整え、完成度を高める。

### Tasks
1. **スタイリング (CSS)**
   - ポケモンカードらしいデザイン (枠線、背景色、ホバーエフェクト) をCSSで実装する。
2. **開封アニメーション**
   - パックを開けた時のワクワク感を出すための簡易的なアニメーションを追加する。
3. **エラーハンドリング**
   - API通信失敗時のメッセージ表示などを追加する。
4. **最終確認**
   - 通しで動作確認を行い、完成！
